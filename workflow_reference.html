+

How to run
python3 scripts/run_scout.py
python3 scripts/run_scout.py --age 1
python3 scripts/run_scout.py --age 7 --expanded
run_scout.py shows a cost estimate and asks Y/n before calling Apify. Flags:
--age 1 = today only (cheapest) · --age 7 = last week · --expanded = 10 searches instead of 5
LinkedIn date filter buckets (no arbitrary days):
--age 1 → past-24-hours  ·  --age 2–7 → past-week  ·  --age 8–30 → past-month
For 2–3 day coverage: run --age 1 each day — cache prevents double-charging.
Cache management
python3 scripts/apify_cache.py status # see what's cached and age python3 scripts/apify_cache.py clear --old # remove stale entries only python3 scripts/apify_cache.py clear # force completely fresh scrape
What happens automatically
run_scout.py checks cache first — same-day re-run costs $0.00
Apify scrapes LinkedIn (5 searches default, 10 if --expanded)
enrich_jobs.py extracts salary, work mode, experience · sorted newest first
score_job skill scores each JD 0–100 against your profile
3-signal duplicate check against all 52+ existing entries
Shortlisted (≥75) added · Review Needed (60–74) flagged · <60 rejected
Google Sheet updated automatically via sheets_sync push

Key columns to check
Col H — Fit ScoreFocus on ≥ 75. 60–74 = Review Needed
Col I — SalaryUpper range > £85k qualifies even if lower end is below
Col J — Work ModeRemote / Hybrid (Xd/week) / On-site
Col K — JD linkClick "View JD" to read full posting on LinkedIn
Critical edit order — do NOT reverse:

1. Open LinkedIn URL → click Apply → copy the ATS redirect URL
2. Paste URL into Col M (career_page_url) — dropdown shows EASY_APPLY option
3. Then set Col L (status) to Approved via dropdown

For Easy Apply jobs: enter EASY_APPLY in Col M instead of a URL
Pull your edits back to the local tracker
